## 中文核心规则

**所有输出必须使用简体中文，无例外。**

适用范围包括但不限于：
- 对话回复
- 工具调用结果说明
- 生成的代码注释
- 文档内容（README、设计文档等）
- 错误信息解释
- 调试分析过程
- 提示词生成结果
- Git 提交信息

即使用户使用英文或其他语言提问，也必须使用简体中文回复。

### 英文使用边界（强约束）
仅以下内容允许保留英文原文：
1. 代码语法（关键字、变量名、函数名、类名等）
2. 终端命令（如 `npm install`、`git status`）
3. 配置项 / JSON 字段 / API 字段名
4. 第三方库、框架、协议名称（如 React、Docker）
5. 报错原文（必须附带中文解释）

除以上情况外，**禁止输出整段英文或中英混杂说明**。

## 工具调用输出

所有工具执行后的**结果描述、摘要说明、状态反馈必须使用中文**：

适用工具包括但不限于：
- 文件操作：`read_file`、`write_file`、`edit_file`
- 代码搜索：`codebase_search`、`grep`
- 终端执行：`run_terminal_cmd`
- 数据/任务工具：`todo_write`
- 网络工具：`web_search`
- 其他所有系统工具

### 输出要求
1. 必须明确说明操作对象 + 操作结果 + 关键数据
2. 不得使用英文提示语（如 Successfully / Failed）
3. 若涉及错误，必须：
    - 保留必要英文错误原文
    - 提供清晰中文解释
    - 给出修复建议（如适用）

### 示例

- ✅ `已成功读取文件 config.json，共 15 行`
- ✅ `命令执行失败：未找到 node 命令（可能未安装 Node.js）`
- ❌ `Successfully executed command`
- ❌ `Error occurred`

## 代码生成与修改规范

### 基本要求
1. 所有代码注释必须使用中文
2. 所有代码解释必须使用中文
3. 不得生成纯英文注释代码

### 修改代码时（强制）
必须包含中文说明：
- 修改了什么
- 为什么修改
- 影响范围
- 是否存在风险

### 解释代码时必须包含：
- 功能说明
- 输入 / 输出
- 核心逻辑
- 注意事项（如有）

---

## 报错与调试规范

1. 保留必要英文报错原文
2. 必须提供中文解释（说明本质原因，而非直译）
3. 必须提供解决思路（步骤化说明）

### 推荐结构
- 错误原因
- 可能触发场景
- 解决方案

---

## Git 提交规范

生成 git commit message 时必须遵循：

### 规则
- 使用**全中文**
- 不超过 **15 字**
- **必须准确描述实际修改内容**
- 禁止空泛、重复或推测性描述

### 类型前缀（必须使用中文）：
- `新增`
- `修复`
- `优化`
- `重构`
- `移除`
